# Skill 打磨报告：wechat-content-router

## 结论

这个 skill 的正确公共化方向，不是继续把“小红书导入”越做越窄，而是上提成一个“微信入口母 skill”。

## 为什么要上提

真实用户的入口是微信，不是单一平台链接。

因此公共版必须具备：

1. 统一配置
2. 统一微信扫描入口
3. 路由分发
4. 子 importer 可替换

## 当前打磨结果

- 已收敛为母 skill：`wechat-content-router`
- 已内置两条高频路由：`xhs` / `mp`
- 已移除必须写死个人 vault 路径的设计
- 已把微信数据库路径、状态文件、导入目录做成配置项

## 下一轮可继续打磨

- 挂接 `quark` / `baidu` 路由
- 增加失败注册表与重试策略
- 增加更完整的 demo 与 CI 验证
